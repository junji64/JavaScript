HTML DOM 을 가지고, 노드 관계(node relationships)를 사용하여 노드 트리(node tree)를 탐색 할 수 있습니다.

DOM 노드(Nodes)

W3C HTML DOM 표준에 따르면, HTML 문서의 모든 것은 노드이다 :

  • 전체 document 는 document 노드이다
  • 모든 HTML 요소는 요소 노드이다
  • HTML 요소 내부의 텍스트는 텍스트 노드입니다
  • 모든 HTML 속성(attribute)은 속성 노드이다
  • 모든 주석(comments) 은 주석 노드이다.

HTML DOM 으로, 노드 트리의 모든 노드는  JavaScript 에 의해 접근 될 수 있다.

새로운 노드가 생성 될 수 있고, 모든 노드가 수정 또는 삭제 될 수 있다.

노드 관계

노드 트리의 노드는 서로에 대해 계층적 관계(hierarchical relationship)를 가진다.

parent, child, 그리고 sibling 이라는 용어가 관계를 설명하는 데 사용됩니다.

  • 노드 트리의 최상위 노드는 root (또는 root 노드) 라고 부른다.
  • 모든 노드는  root (부모가 없는)를 제외하고 정확히 하나의 parent 를 가진다.
  • 노드는 여러 children 를 가질 수 있습니다
  • Siblings (brothers or sisters)은 같은 parent 를 가진 노드입니다
<html>

  <head>
      <title>DOM Tutorial</title>
  </head>

  <body>
      <h1>DOM Lesson one</h1>
      <p>Hello world!</p>
  </body>

</html>

위의 HTML은 다음과 같이 읽힐 수 있습니다 :

  • <html> 이 root 노드
  • <html> 는 부모(parents)가 없음
  • <html> 은 <head> 와 <body>의 부모(parent)
  • <head> 는 <html> 의 첫 번째 자녀(child)
  • <body> 는 <html> 의 마지막  자녀(child)

and:

  • <head> 는 한 자녀(child) <title> 를 가진다
  • <title>  는 한 자녀(child) (a text node): "DOM Tutorial" 를 가진다.
  • <body> 는 두 자녀(children): <h1> 과 <p> 를 가진다
  • <h1> 는 한 자녀(child): "DOM Lesson one" 를 가진다
  • <p> 는 한 자녀(child): "Hello world!" 를 가진다
  • <h1> 과 <p> 는 형제(siblings) 이다.

Nodes 들 사이의 탐색

JavaScript 로 노드들 사이를 이동하기 위해 다음과 같은 노드 속성(properties)을 사용할 수 있습니다 :

  • parentNode
  • childNodes[nodenumber]
  • firstChild
  • lastChild
  • nextSibling
  • previousSibling

Warning !

DOM 처리에서 흔한 오류는 텍스트를 포함하는 요소 노드를 예상하는 것입니다.

다음 예에서: <title>DOM Tutorial</title>,  요소 노드 <title> 은 텍스트를 포함하지 않는다. 대신에 "DOM Tutorial" 이라는 값을 가진 text node 를 갖는다

text  노드의 값은 노드의 innerHTML 속성(property), 또는 nodeValue 로 접근 될 수 있다.

자식(Child) 노드와 노드 값

innerHTML 속성 이외에, 요소의 내용을 얻기 위해 childNodes 와 nodeValue를 속성을 사용할 수 있습니다.

다음 예에서는 id="intro" 인 <h1> 요소의 노드 값을 가져다가 <p> 요소에 복사합니다:

Example

<html>
<body>

<h1 id="intro">My First Page</h1>

<p id="demo">Hello!</p>

<script>
var myText = document.getElementById("intro").childNodes[0].nodeValue;
document.getElementById("demo").innerHTML = myText;
</script>

</body>
</html>
Try it Yourself »

위의 예에서, childNodes 와 nodeValue 는 속성(properties)이지만, getElementById 는 메서드입니다.

이 튜토리얼에서 우리는 innerHTML 속성(property)을 사용합니다. 그러나 위의 방법을 배우는 것은 트리 구조와 DOM의 탐색을 이해하는 데 유용합니다.

firstChild 속성의 사용은 childNodes[0] 의 사용과 동일합니다.:

Example

<html>
<body>

<h1 id="intro">My First Page</h1>

<p id="demo">Hello World!</p>

<script>
myText = document.getElementById("intro").firstChild.nodeValue;
document.getElementById("demo").innerHTML = myText;
</script>

</body>
</html>
Try it Yourself »

DOM Root Nodes

전체 문서(document)에 접근할 수 있는 두 가지 특별한 속성(properties) 이 있습니다.:

  • document.documentElement - 전체 문서(document)
  • document.body - 문서(document) 의 본문(body)

Example

<html>
<body>

<p>Hello World!</p>
<div>
<p>The DOM is very useful!</p>
<p>This example demonstrates the <b>document.body</b> property.</p>
</div>

<script>
alert(document.body.innerHTML);
</script>

</body>
</html>

nodeName 속성(Property)

nodeName 속성(property)은 노드의 이름을 지정합니다.

  • nodeName 은 읽기 전용(read-only) 입니다
  • 요소 노드의 nodeName 은 태그 이름과 동일합니다
  • 속성(attribute) 노드의 nodeName 은 속성(attribute)의 이름입니다
  • 텍스트 노드의 nodeName 은 항상 #text 입니다.
  • 문서(document) 노드의 nodeName 은 항상 #document 입니다.

Note: nodeName 은 항상 HTML 요소의 대문자 태그 이름이 포함되어 있습니다.

nodeValue 속성(Property)

nodeValue 속성(property)은 노드의 값을 지정합니다.

  • 요소 노드의 nodeValue 는 정의되지 않습니다.
  • 텍스트 노드의 nodeValue 는 텍스트 자체입니다.
  • 속성(attribute) 노드의 nodeValue 는 속성(attribute) 값 입니다.

nodeType 속성(Property)

nodeType 속성은 노드의 타입을 반환합니다. nodeType 은 읽기 전용(read only) 입니다..

가장 중요한 node types 들은 다음과 같습니다.:

Element type NodeType
Element 1
Attribute 2
Text 3
Comment 8
Document 9
